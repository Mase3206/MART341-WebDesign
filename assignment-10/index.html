Good intentions

I have always wanted to experiment with cluster computing. However, I haven't had the time or resources to do such a thing — until now. It turns out that used Lenovo ThinkCentre M900s are very affordable.

I purchased three of those for around $250, which is actually very affordable. For some reason, most listings for computers like these are "barebones", meaning they don't include a CPU, RAM, or any storage, and the list price is often very similar to fully-populated machines. Even if they are cheaper, the cost of the barebones machines plus the missing components will likely be over $150 per device.

I will be using Proxmox Virtual Environment (VE) 8.0 as the VM hypervisor and cluster manager. Here's what I will run in it:

  • A graphical file server using OwnCloud, which is basically self-hosted Google Drive
  • A Transmission BitTorrent client
  • A HandBrake instance
  • A simple nginx server for reverse proxy

These will all run in Docker containers (as much as I dislike them) in a Fedora Cloud 38 VM. You could run them in any OS you like, but Fedora is just my preference, and they have a version designed specifically for running in a VM.

A fourth node?

I might also add my gaming PC to the cluster. This has some benefits, such as providing a node with a GPU for video encoding and decoding as well as general compute. It would also allow me to actually expose the Docker containers directly to the network. There are ways to add more devices to each Ethernet port in my dorm room, such as via a network switch, but I don't possess the knowledge to do so in a way that would render it undetectable.

If I do end up adding my gaming PC to the cluster, I would run a couple more Docker containers in the Fedora Cloud VM:

  • A Plex server. I'm currently running it on my gaming PC in Windows, but I would instead run it in a Docker container under Fedora.
  • A Samba Server. Again, I'm currently running this on my PC. This and OwnCloud are critical to the server, as I need to be able to add and remove files to and from the storage in my PC.

My gaming "PC" would become a Windows 10 virtual machine with direct, hardware-level access to the GPU and Bluetooth dongle. I connect to it remotely via Parsec anyways, so I won't be losing anything if I am no longer able to play on a monitor connected directly to the device.
