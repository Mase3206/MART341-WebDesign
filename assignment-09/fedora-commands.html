Common Fedora commands

  • cd [path]
    Changes the current directory of the terminal. If called without a file path, it will set the current directory to the user's home folder at /home/(username). 
  • ls [options] [path]
    Lists the contents of a specified directory. If called without a file path, it will list the contents of the current directory.
    Common Options: 
    • a
      Lists all files, folders, and symlinks, including those that are hidden. 
    • l
      Lists the permissions, owner user & group, and size in bytes of files. (aka long) 
    • h
      Converts file sizes into a human-readable format. 
  • nano [file]
    A simple text editor. To open a file, call nano with a file. If that file does not exist, nano will make it. To save it, type Ctrl+O. To exit, type Ctrl+X. 
  • dnf [command]
    The built-in package manager for REHL-based distributions, like Fedora, RedHat, and CentOS (R.I.P.). Requires root-level permissions — use sudo.
    Note that dnf is used only in REHL-based distros. Debian-based distros use apt, while Arch-based distros use pacman and yay.
    Common Commands: 
    • update
      Fetches package lists from the Internet. If any newer packages are available, dnf will download and install them. 
    • install <package(s)>
      Fetches package lists from the Internet. If package(s) exist and are not already installed (or can be updated), dnf will download and install them. 
    • remove <package(s)>
      If packages(s) exist and are installed on the system, dnf will remove them and their unused dependencies. 
  • ip address
    Lists the full information for all the network interfaces that exist on the system. ip (and networking in general) confuses the heck out of me, so the Internet is your friend here.
