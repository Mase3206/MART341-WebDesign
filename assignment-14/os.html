TrustyCentre

Software Hardware Networking OS

The TrustyCentre contains multiple devices within it: three Lenovo ThinkCentres, one dedicated computer as the NAS, one Raspberry Pi 3, one Raspberry Pi Pico, and one router. This menagerie of technology requires multiple flavors and kinds of OSes.

ThinkCentres

The ThinkCentres all run Rocky Linux 9, the open source variant of Red Hat Enterprise Linux (RHEL, for short). I previously planned on using Fedora Server, the upstream version of RHEL (and thus Rocky), but its release cycle is far too fast for this particular project.

NAS

Currently, the interim NAS is my gaming PC with a VM and a couple Samba shares. This is not the long-term plan, but it will remain for the time being. The long term plan is to repurpose my previous gaming PC once I get a new one as a TrueNAS server. It will only be serving files and acting as the occasional PXE boot and network provisioning server.

Raspberry Pis

The Raspberry Pi 3 runs a flavor of Debian Linux called Raspberry Pi OS (formerly called Raspbian, R.I.P.). This is a super lightweight operating system designed specifically for the Raspberry Pi family of ARM-based, single board computers.

The Raspberry Pi Pico doesn't technically run an operating system, as it is a microcontroller, not a computer. Micorcontrollers are great at doing simple, repetitive tasks very reliably. In my case, the Pico is a simple PWM fan controller with automatic speed adjustments using temperature measurements from various thermal probes. Three probes are placed just next to each ThinkCentre power supply, with a fourth placed in the PDU shelf just below the power supplies.

The two Raspberry Pis operate together. The Pi 3 hosts a simple web interface to view and change fan speeds while the Pico does the actual controlling of fans. To make this work, the two are in constant communication via a discrete protocol over UART. I wanted to use SPI, but Raspberry Pis can only operate in Host mode, and I would need the Pico to operate in Client mode.

The router is a GL.inet travel router running OpenWRT-based firmware. This is pretty close to what I need and want in a super compact package. The router facilitates the connection between the ThinkCentres and Raspberry Pis and the larger network — though, when I have an apartment and am managing my own Internet, it will probably just be the router, not just the gateway after the router. I cover this in more detail on the networking page.
